Why should I trust you?

This site does not store or transmit any data you enter into it. The passwords it generates are extremely expensive to crack by brute force (compared to passwords most people use) and using them means that if an attacker steals one site’s database, your other accounts are safe.

How does it work?

It concatenates your passphrase, a fixed UUID value and the service name you type in, and calculates the SHA-256 hash of that string. It then encodes the bits of this hash using a 92-character alphabet, subject to the given character constraints. This scheme makes sure that a small change to the input makes a big change to the output, and it means the generated password has a very high degree of entropy.

Is there a command line version?

Of course there is. If you have Node installed, run npm install -g vault. You’ll also want to take a look at the readme.
